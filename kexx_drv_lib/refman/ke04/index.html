Introduction

This document describes the Freescale's software drivers for Kinetis KE family of microcontrollers. These software drivers are provided in a source code form. The source code files are intended to be included directly into the application project or built separately into as a statically linked library. This document should give you all information about drivers' configuration and its use in your application. The Directory Structure explains the source files and header files important for your application. The Drivers FAQ is available to answer the most common questions. Please read the License document for terms and conditions under which you can use the software.

CREATE NEW PROJECT
Execute make_new_project_<device>.exe script under the "build/<toolchain>/<device>" subfolder to create a new project for <device>, where <toolchain> is iar for IAR Embedded Workbench, keil for Keil uVision, or cw for CodeWarrior; <device> is KE02, KE04, and KE06. The script creates new project based on the selected toolchain, debugger probe, device derivative and project name. The new project will contain files with all necessary modifications.

BASIC DATA TYPES
The drivers use CMSIS data types to support code portability between different hardware architectures and tools. These data type definitions can be found in the stdint.h header file. Click Basic Data Types for the similar version.

CORE'S PLATFORM INTERFACE
The drivers use CMSIS core API to interface to the Kinetis KE core's platform architecture based on the ARM's Cortex-M0+ core. Refer to ARM website for more details.

PERIPHERAL DRIVERS
Freescale's Kinetis KE devices are integrated with 12-bit analogue to digital converter with FIFO, high speed comparator, timers, communication interfaces and additional on-chip peripherals.
