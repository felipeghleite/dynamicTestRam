Directory Structure

The current version of the peripheral drivers' project has the following directory structure:

  • [trunk] - Main installation directory
    • [build] - Project subfolder
      • [cw] - CodeWarrior Project Folder
        • [ke06] - Target Device folder
          • [projects] - Example projects
      • [iar] - IAR EWARM Project Folder
        • [ke06] - Target Device folder
          • [projects] - Example projects
          • make_new_project_ke06.exe - Run this script and enter input arguments to create new project
      • [keil] - KEIL Project Folder
        • [ke06] - Target Device folder
          • [projects] - Example projects
    • [refman] - Reference manual subfolder
      • [drivers] - Drivers documentation
    • [src] - Source code repository
      • [common] - Common source code
      • [cpu] - CPU access code
      • [drivers] - Drivers source code
        • [acmp] - ACMP driver code
        • [adc] - ADC driver code
        • [bitband] - BITBAND driver code
        • [bme] - BME driver code
        • [can] - CAN driver code
        • [crc] - CRC driver code
        • [ftm] - FTM driver code
        • [gpio] - GPIO driver code
        • [ics] - ICS and OSC driver code
        • [iic] - IIC driver code
        • [kbi] - KBI driver code
        • [nvm] - NVM driver code
        • [pit] - PIT driver code
        • [pmc] - PMC driver code
        • [pwt] - PWT driver code
        • [rtc] - RTC driver code
        • [sim] - SIM driver code
        • [spi] - SPI driver code
        • [uart] - UART driver code
        • [wdog] - WDOG driver code
      • [projects] - Project source code folder
        • [ke06] - Target Device
          • [demos] - Example source code

The source code for drivers and examples are located in the drivers and projects subfolders. The project files are placed in a subfolder for the respective compiler of the build folder.
